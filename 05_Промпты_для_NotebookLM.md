# Промпты для глубокого исследования (NotebookLM)

> Закрывает: «подготовка промптов для deep research в NotebookLM».

## Как этим пользоваться (важно про NotebookLM)
NotebookLM отвечает ТОЛЬКО по источникам, которые ты в него загрузил (PDF, ссылки, Google Docs, YouTube). Он не ищет в открытом интернете сам. Поэтому стратегия двойная:
1. **Сбор источников.** Сначала найди и загрузи документы (через обычный поиск/Gemini с веб-поиском, по ссылкам из `Sources.xlsx`, сайты RIS, EUAA, BVwG). 
2. **Допрос источников.** Потом задавай NotebookLM промпты ниже — он синтезирует, цитирует и не выдумывает.

Для поиска самих источников (где взять решения судов и т.д.) используй промпты из раздела C в Gemini/Perplexity с веб-поиском, а найденное грузи в NotebookLM.

Промпты сгруппированы: A — карта правового поля, B — наполнение RAG-контента, C — поиск судебных решений (твоя отдельная задача), D — генерация тест-вопросов.

## A. Карта правового поля (для раздела «Market & Technical Discovery», 15%)

**A1. Структура иммиграционного права Австрии для беженцев из Беларуси**
```
Ты — эксперт по миграционному праву Австрии. На основе загруженных источников составь структурированную карту правового поля для гражданина Беларуси, ищущего убежище или легализацию в Австрии. Раздели по темам: (1) подача на убежище (Asyl), (2) виды ВНЖ (Aufenthaltstitel), (3) гражданство (Staatsbürgerschaft), (4) право на работу, (5) воссоединение семьи. По каждой теме укажи: ключевые законы и параграфы (NAG, FPG, AsylG, StbG), ответственные органы, типичные сроки, типичные основания отказа. Везде давай точную ссылку на источник и параграф. Где источники расходятся или информация устарела — отметь это явно.
```

**A2. Глоссарий терминов (RU ↔ DE) — критично для cross-lingual RAG**
```
Составь двуязычный глоссарий (русский — немецкий) ключевых юридических терминов из загруженных документов по теме убежища и легализации в Австрии. Для каждого термина: немецкий оригинал, русский перевод, краткое объяснение простым языком, и параграф/закон, где он определён. Включи термины: Aufenthaltstitel, Niederlassungsbewilligung, Rot-Weiß-Rot Karte, subsidiärer Schutz, Asylberechtigter, Aufenthaltsberechtigung, Niederlassungsnachweis, Fremdenpolizei. Выведи таблицей.
```
> Этот глоссарий загрузи потом в свой RAG как отдельный документ — он напрямую улучшит cross-lingual retrieval.

## B. Наполнение и обогащение RAG-контента

**B1. Сжатие закона в FAQ-формат (для contextual retrieval и few-shots)**
```
На основе загруженного текста закона [NAG / FPG / StbG] сформулируй 15 пар «вопрос беженца — точный ответ». Вопросы — простым языком, как их задал бы обычный человек (на русском). Ответы — строго по тексту закона, с обязательной ссылкой на конкретный параграф (§) и без додумывания. Если закон не даёт прямого ответа — так и напиши. Формат: Вопрос / Ответ / § источника.
```

**B2. Контекстные аннотации к разделам (под contextual retrieval)**
```
Для каждого параграфа загруженного закона напиши одно предложение-аннотацию на русском: о чём этот параграф и в какой жизненной ситуации он применяется. Формат: «§ X — [аннотация]». Это нужно как метаданные для поисковой системы.
```

## C. Поиск судебных решений (твоя отдельная задача — где брать прецеденты)
> Эти промпты — для Gemini/Perplexity с веб-поиском, чтобы НАЙТИ источники. Найденное грузи в NotebookLM.

**C1. Где искать решения австрийских судов по миграции**
```
Найди официальные и бесплатные базы данных решений австрийских судов по миграционным и убежищным делам. Меня интересуют: Bundesverwaltungsgericht (BVwG), Verwaltungsgerichtshof (VwGH), Verfassungsgerichtshof (VfGH). Для каждого источника укажи: URL, как фильтровать по теме «убежище/Asyl» и «гражданство Беларуси», доступен ли полный текст решений, есть ли API или экспорт, язык. Также проверь базы RIS (ris.bka.gv.at) и EUAA Case Law Database. Дай прямые ссылки на разделы фильтрации.
```

**C2. Релевантные прецеденты по беларусам**
```
Найди и кратко изложи публично доступные решения австрийских судов (BVwG/VwGH) по делам об убежище граждан Беларуси за 2020-2026 годы. Для каждого: номер дела, дата, суть, исход, ключевой правовой аргумент, ссылка на полный текст. Отметь, какие можно скачать целиком для дальнейшего анализа.
```

## D. Генерация тест-вопросов для evaluation (расширение golden set)

**D1. Синтетические тест-вопросы из корпуса**
```
На основе ТОЛЬКО загруженных документов сгенерируй 40 тестовых пар «вопрос-эталонный ответ» для оценки юридического ассистента. Требования: вопросы реалистичные, на русском, разной сложности (простые фактические, многошаговые, пограничные). Каждый эталонный ответ — строго из источника, с указанием § и закона. Добавь 5 «вопросов-ловушек», на которые в документах НЕТ ответа (правильная реакция системы — честно сказать «недостаточно информации»). Формат таблицы: Вопрос | Эталонный ответ | Источник(§) | Тип | Есть_ли_ответ(да/нет).
```
> «Вопросы-ловушки» — золото: они проверяют, не галлюцинирует ли бот. Преподаватель это оценит.

**D2. Вопросы на cross-lingual устойчивость**
```
Переведи следующие 13 русских тестовых вопросов на белорусский язык, сохранив юридический смысл. Цель — проверить, одинаково ли хорошо система отвечает на русском и белорусском. Выведи таблицу: Русский | Белорусский.
```

## Что НЕ делать с NotebookLM
- Не проси его «найти в интернете» — он работает по загруженным источникам.
- Не доверяй ответу без ссылки — всегда требуй цитату источника (он это умеет).
- Не загружай туда персональные данные людей.
